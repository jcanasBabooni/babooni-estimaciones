Acciones Ideauto

Decisiones, entregables e infra fuera del alcance de desarrollo.

  • Bloqueante B-1 · Login
    Estandar de secretos / Key Vault Propuesta de un estandar para el login y la gestion de secretos. Estandar: Azure Key Vault + Managed Identity Implantacion a cargo de Babooni. Recursos minimos (Key Vault)
    • 1 Resource Group.
    • 1 Azure Key Vault.
    • 2 App Registrations (o 2 Managed Identities).
    • Asignar el rol Key Vault Secrets User a cada aplicacion.
    • Guardar los secretos en el Key Vault.
    • Que las aplicaciones usen el SDK de Azure (DefaultAzureCredential o ClientSecretCredential) para leer los secretos.
    Alternativa sin Key Vault
    • Si Ideauto concede permisos sobre las pipelines de Azure, los secretos se pueden gestionar desde Azure Pipelines / Variable Groups / Pipeline secrets.
    • En ese caso no haria falta Key Vault para sacar los secretos del codigo (inyeccion en despliegue, sin hardcode).
    • Pendiente Ideauto: otorgar esos permisos de pipelines a Babooni.
  • Certificado
    Certificado (entregable Ideauto)
    • Responsable: Antonio (Ideauto).
    • Entrega: el certificado debe proporcionarlo Ideauto a Babooni.
    • Dependencia: sin ese entregable no se puede avanzar el tramo dependiente (TDE / cifrado).
  • Infra
    WAF / firewall entre app y cliente

    Seguridad en capa 7 (HTTP/HTTPS). Analiza cada peticion entrante antes de que toque los servidores y bloquea trafico malicioso.

    • Responsable: Ideauto (infra / plataforma). Contexto: Interempresas.net.
    • Proteccion de APIs: al ser una plataforma abierta a marcas, energeticas y concesionarios, el WAF analiza que las peticiones a la API no contengan SQL Injection, Cross-Site Scripting (XSS) u otras vulnerabilidades OWASP Top 10.
    • Rate limiting: evita que un script defectuoso de un concesionario o un ataque tumbe el servidor (p. ej. miles de peticiones por segundo).
    • Filtro de bots y fraude: bloquea scrapers o bots que intenten consultar datos masivos de matriculaciones.
    • Accion: despliegue y operacion del WAF entre la aplicacion y el cliente.
